Aiti Guru. Тестовое задание 
Привет! Основной фронтенд проект нашей команды - это административная система управления, насыщенная таблицами, инструментами для редактирования и сложными формами. Поэтому тестовое задание максимально приближено к нашим ежедневным задачам.

Мы в своей работе в основном работаем с сущностями товаров, поэтому перед тобой макет похожего интерфейса. Тебе необходимо реализовать приложение, которое загрузит список товаров и позволит пользователю удобно взаимодействовать с данными: сортировать, редактировать и добавлять новые позиции. 
Условия:
Используй Vite. https://vite.dev/.
Используй React.
Используй TypeScript (строгая типизация обязательна).
Работоспособность в актуальной версии Google Chrome.
Работа с формами https://react-hook-form.com/.
UI библиотека shadcn. https://ui.shadcn.com/docs, https://ui.shadcn.com/docs/installation/vite
Стейт-менеджер Zustand. https://zustand.docs.pmnd.rs/learn/getting-started/introduction
Остальное на твоё усмотрение, но будь готов обосновать выбор.

Дизайн:
Реализация должна визуально соответствовать макету в Figma. https://www.figma.com/design/MbyxPG0ZgG9NhFGVq56xMt/Aiti-Guru-%D1%82%D0%B5%D1%81%D1%82%D0%BE%D0%B2%D0%BE%D0%B5-%D0%B7%D0%B0%D0%B4%D0%B0%D0%BD%D0%B8%D0%B5?node-id=1-299&t=yrb5pgyibSdh1Yex-0

API:
В качестве источника данных использовать публичное API: DummyJSON Products. https://dummyjson.com/docs/products
Для авторизации использовать: DummyJSON Auth. https://dummyjson.com/docs/auth

Функциональные требования:
Форма входа:
Валидация полей (обязательность заполнения).
Обработка ошибок: если API возвращает ошибку, выводить уведомление или текст ошибки под полями.
Логика запоминания данных для входа:
Если чекбокс установлен, нужно сохранять токен авторизации так, чтобы сессия жила после закрытия браузера
Если не установлен, то сессия должна сбрасываться при закрытии вкладки
Вывод списка товаров:
Соответствие столбцам из макета Figma
Прогресс-бар при подгрузке
Подгрузка данных из API.
Сортировка:
Возможность сортировки по столбцам (например, по цене или рейтингу).
Должно храниться состояние сортировки 

Добавление товара:
По нажатию кнопки "Добавить" открывается форма добавления товара с возможностью заполнить основные поля: Наименование, цена, вендор, артикул.
При успешном добавлении  показывать базовое Toast уведомление 
Логику сохранения через API делать при этом не нужно.
Логика интерфейса:
Если рейтинг товара ниже 3, значение должно подсвечиваться красным цветом.
Поиск товаров:
Использовать API
